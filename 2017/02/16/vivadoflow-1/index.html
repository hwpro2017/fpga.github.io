Vivado上手教程1-创建工程

发表于 2017-02-16 | 分类于 教程 | | 浏览 次

如何创建Vivado工程？
关于创建Vivado工程你所应该知道的。。

Vivado创建工程

按照上一节中所述，任选一种姿势打开Vivado

如果姿势正确，这是玩家们可以看到如下Vivado界面。

在界面中，分有三行三列图标可点：

  1. Quick Start ：
  • Create New Project 创建工程 （本节内容）
  • Open Project 打开工程 （常用）
  • Open Example Project 打开案例工程（常用）
  1. Tasks：
  • Manage IP 管理IP （后续博文详细介绍）
  • Open Hardware Manager 硬件管理器（后续博文详细介绍）
  • Xilinx Tcl Store 脚本库（后续博文详细介绍）
  1. Information Center：
  • Documentation and Tutorials 文档中心（常用）
  • Quick Take Videos 视频中心（不常用）
  • Release Notes Guide 通知中心（不常用）

创建工程

首先点击Quick Start下的Create New Project

这一页没啥可选的，直接NEXT；

填写工程路径和工程名称

需要注意的是：一定不能有中文字符和空格，此处建议勾选“Create project subdirectory”，继续点击NEXT；

选择工程类型

  • RTL Project ：通常基于VHDL, Verilog, SystemVerilog，EDIF网表为源文件时选择此项工程类型；（大多数情况）
  • Post-synthesis Project ：基于已综合后的网表创建工程；
  • I/O Planning Project ： 用于做IO规划的工程，可以没有源文件；
  • Imported Project ：导入ISE或其他第三方工具如Synplify的工程，注意导入的仅仅只有源文件和编译顺序，而不会导入实现结果与工程设定；
  • Example Project ： 打开案例工程。

通常一般情况，我们选择RTL型工程进行创建，此处建议勾选“Do not specify sources at this time”，可以不在创建新工程向导中进行源文件导入；完成后继续点击NEXT；

选择目标器件

通过筛选器来快速指定器件，或通过板卡”Boards”选型通过板卡来快速指定工程目标；
注意，”Boards”选项中默认仅有官方自有板卡可选，第三方或玩家们自制板卡可自定义板卡定义文件后存入Vivado指定目的，即可再此处快速选择；完成选型后，点击NEXT；
（关于如何自定义板卡定义文件，有兴趣的玩家们可以联系博主）

完成工程创建

最终，确认工程创建信息，无误后，点击Finish完成工程创健。
